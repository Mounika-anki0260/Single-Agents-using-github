# Configuration and Deployment

## Build tools and commands

- Ant / NetBeans project: `build.xml` under project folders (e.g., `Project/TurkishAirlines/build.xml`). Use NetBeans or `ant` to build a WAR.

Example Ant commands (run from project folder):

```bash
ant clean
ant build
ant dist
```

## Deployment

- Expect a WAR artifact generated by Ant; deploy to a servlet container (Tomcat, GlassFish).
- No Dockerfile or Kubernetes manifests found — containerization not present by default.

## Environment / runtime assumptions

- Java runtime (JDK 8+ recommended, confirm with project source and libraries).
- JDBC driver for Access (if used) or for the production DB (Postgres) must be available in the runtime classpath.
